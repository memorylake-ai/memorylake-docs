## Authentication Issues

<AccordionGroup>

<Accordion title="401 Unauthorized error">
**Causes**: Invalid or missing API key

**Solutions**:
- Verify you're using the full API secret (not masked version)
- Check Authorization header format: `Bearer YOUR_SECRET`
- Ensure no extra spaces or newlines in key
- Verify key hasn't been deleted
- Check for typos when copying key
- Generate new key if original lost
</Accordion>

<Accordion title="403 Forbidden error">
**Causes**: Valid key but no access to resource

**Solutions**:
- Verify project ID in URL is correct
- Check key is for the correct project
- Ensure project still exists
- Verify API endpoint URL is correct
</Accordion>

<Accordion title="Lost API secret">
**Solution**: Secrets cannot be retrieved. Delete the API key and create a new one.
</Accordion>

</AccordionGroup>

## API Request Issues

<AccordionGroup>

<Accordion title="Network timeout">
**Solutions**:
- Check internet connection
- Verify API endpoint URL is correct
- Try request again (temporary server issue)
- Check if service status page shows outages
- Increase request timeout in your code
</Accordion>

<Accordion title="500 Internal Server Error">
**Solutions**:
- Wait a few minutes and retry
- Check MemoryLake status page
- Verify request format is correct
- Contact support if persistent
</Accordion>

<Accordion title="Rate limit exceeded">
**Solutions**:
- Reduce request frequency
- Implement exponential backoff
- Cache responses when possible
- Contact support for higher limits
</Accordion>

</AccordionGroup>

## Configuration Issues

<Accordion title="Environment variable not loading">
**Solutions**:
- Verify `.env` file exists in correct location
- Check file name is exactly `.env`
- Ensure `.env` is loaded before code runs (e.g., `require('dotenv').config()`)
- Check for typos in variable names
- Restart application after changing `.env`
</Accordion>

<Accordion title="Key works locally but not in production">
**Solutions**:
- Verify environment variables set in production
- Check production is using correct `.env` file
- Ensure production deployment includes environment setup
- Verify no hardcoded dev keys in production
</Accordion>

</AccordionGroup>

## Getting Help

If issues persist:

1. **Gather Information**:
   - Exact error message
   - HTTP status code
   - API endpoint being called
   - Request headers (remove API key before sharing!)
   - Response body

2. **Check Documentation**:
   - [API Reference](/features/memorylake/api-reference/overview)
   - [Authentication Guide](/features/memorylake/api-reference/authentication)

3. **Contact Support**:
   - Include gathered information
   - Describe what you expected vs. what happened
   - Note any recent changes
