## Overview

MemoryLake's tree view provides an intuitive interface for browsing your document hierarchy. Learn how to navigate efficiently with lazy loading, keyboard shortcuts, and visual organization features.

## Tree View Interface

{/* SCREENSHOT: Full tree view showing hierarchical structure */}

The tree view consists of several key components:

- **Tree Panel**: Left sidebar showing your folder hierarchy
- **Expand/Collapse Icons**: Click to show or hide folder contents
- **File/Folder Icons**: Visual indicators for item types
- **Resizable Divider**: Drag to adjust tree panel width
- **Main Content Area**: Shows details of the selected item

## Navigation Basics

### Expanding and Collapsing Folders

**Mouse Navigation**:
- Click the arrow icon next to a folder to expand it
- Click again to collapse
- Click the folder name to select without expanding

**Keyboard Navigation**:
- `‚Üí` (Right Arrow): Expand selected folder
- `‚Üê` (Left Arrow): Collapse selected folder
- `‚Üë` (Up Arrow): Move to previous item
- `‚Üì` (Down Arrow): Move to next item
- `Enter`: Expand folder or open file

{/* SCREENSHOT: Expanded and collapsed folders in tree */}

<Tip>
Use keyboard navigation for faster browsing, especially when working with deeply nested folders.
</Tip>

### Lazy Loading

MemoryLake uses lazy loading to improve performance:

**How It Works**:
1. Folders show a collapse/expand icon but contents aren't loaded initially
2. When you expand a folder, MemoryLake fetches its children from the server
3. A loading indicator appears briefly while contents load
4. Once loaded, the folder's contents remain in memory until you refresh

**Benefits**:
- Faster initial page load
- Reduced memory usage
- Scales to handle thousands of files
- Server fetches only what you need

<Note>
If a folder's contents change (files added/removed), you may need to refresh or re-expand the folder to see updates.
</Note>

## Resizable Columns

### Adjusting Tree Panel Width

The tree panel width is adjustable to fit your workflow:

1. **Locate the Divider**: Find the vertical line between the tree and content area
2. **Hover**: The cursor changes to a resize indicator
3. **Drag**: Click and drag left or right to resize
4. **Release**: The new width is saved

**Width Constraints**:
- **Minimum**: 15% of screen width
- **Maximum**: 50% of screen width
- **Recommended**: 25-30% for most workflows

{/* SCREENSHOT: Resizing tree panel by dragging divider */}

<Tip>
For deeply nested folders with long names, increase the tree panel width. For simple structures, minimize it to see more content.
</Tip>

## Advanced Navigation

### Breadcrumb Navigation

Breadcrumbs show your current location:
- Click any breadcrumb to jump to that folder
- Useful for quickly moving up multiple levels
- Always visible at the top of the content area

Example: `Root > Projects > Research > 2024 > Q1`

### Search and Filter

Find items quickly without manual navigation:

1. Press `Ctrl/Cmd + F` or use the search box
2. Type part of the file or folder name
3. Tree filters to show only matching items
4. Press `Esc` to clear filter and show all items

<Note>
Search filters the visible tree. Folders must be expanded for their contents to be searchable.
</Note>

### Jump to Item

If you know the item's path:
1. Use the "Jump to" or "Go to Path" feature
2. Enter the full path (e.g., `/Projects/Research/paper.pdf`)
3. The tree automatically expands and scrolls to that item

## Performance Tips

### Working with Large Trees

For drives with thousands of files:

- **Don't Expand Everything**: Keep folders collapsed unless needed
- **Use Search**: Find items without expanding every folder
- **Organize Logically**: Group files so you don't need to browse deep hierarchies
- **Lazy Loading**: Trust it - folders load quickly when expanded
- **Refresh Sparingly**: Only refresh when you know content changed

### Memory Management

The tree view manages memory efficiently:
- Collapsed folders unload from memory after inactivity
- Visible items stay cached for instant re-access
- Large folders paginate automatically
- Browser memory is monitored and optimized

<Tip>
If the interface becomes sluggish, collapse deep folder hierarchies or refresh the page to clear the cache.
</Tip>

## Accessibility

The tree view is designed for accessibility:

- **Keyboard Navigation**: Full functionality without a mouse
- **Screen Readers**: ARIA labels for all tree nodes
- **High Contrast**: Works with high contrast modes
- **Focus Indicators**: Clear visual focus for keyboard users
- **Logical Tab Order**: Follows natural reading order
